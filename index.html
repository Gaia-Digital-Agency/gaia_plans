PART A · 1/2Business Plan — The Venturi Model

Gaia AI is a service-enabled tech platform where AI labour scales on GCP, and humans give it thinking soul. The Venturi thesis: channel broad market demand through a narrow AI throat to produce high-velocity outputs — grow revenue without growing headcount.

Target state: 25 AI-assisted orchestrators delivering the work of 75 producers · RM 1M revenue by Month 18 · 20% IRR · recurring > one-off billable.

Service verticals (7)

HospitalityF&B Web DevSEO BrandingAds Social Media

Core targets

RM 1Mrevenue · month 18
25orchestrators
20%internal rate of return
3×output / headcount

Guiding principles

Venturi Compression No-Manpower-Growth Revenue Quality > Volume Coexist Before Cutover Single Source of Truth Human Judgment at the Edges

PART B · 1/2Figma → Site Automation (Templatebase + Victor)

Design in Figma, ship a running production site in a fraction of the time. Combines Templatebase (a reusable VRTPN-stack scaffold) with Victor — the Openclaw multi-agent orchestrator (repo openclaw_var) — plus Claude Code for build work and a Python QA engine for validation.

Stack in one line: Figma MCP → Victor (Openclaw) ⇄ Claude Code → Templatebase (Vite SSR + React 19 + Tailwind v4 + Payload CMS + Node + Postgres) → Python QA → GCP.

Components

Templatebase (VRTPN) Victor — Openclaw Agent Claude Code Figma MCP Payload CMS Python QA (7 tools) GCP

Victor's agent roster (openclaw_var)

main (Victor · orchestrator) web-manager (CMS / blog) copywriter writer (email + letters) booking (calendar) drive-manager analyst

1 · Scaffold from Templatebase 2 · Rebrand & isolate 3 · Go live 3.5 · Seed strategy 4 · Figma MCP setup 4.1 · Tokens-first 4.3 · IA rework 4.4 · Home blocks 4.5 · Header/Footer 4.7 · Polish

PART C · 1/2Social Media Automation (SOMA)

Five-platform orchestration where AI drafts content and replies, humans approve before publishing, and Openclaw executes the push. Extends the Openclaw framework with platform-specialist agents.

Core rule: OpenClaw performs every publish — never the AI direct. GDA retains human approval over all AI-generated content.

Platforms

Instagram (Graph API) Facebook (Graph API) YouTube (Data API v3) X (API v2) TikTok (Content Posting)

PART D · 1/2Sample POC — Blossom School Catering + Brian

Proof-of-concept showing Openclaw running a full-working service, not just assisting. Brian (repo openclaw_bsc) is the public-facing WhatsApp AI operator; Orders is the internal execution agent that talks to the Schoolcatering API. Families register, order meals, cancel, and get reminders — all through WhatsApp.

Live runtime: WhatsApp → Brian (openclaw_bsc, gateway :18789 on gda-ai01) → Orders → BSC API https://schoolcatering.gaiada1.online/api/v1. Public URL: bsc.gaiada0.online.
